A different take - DT in sync link 

Ultimately what you are looking for is an extremely solid development life-cycle, and TDD does not give that to you in totality. Breaking it down into four simple steps will get you a long way:
  • Gather requirements & write user stories or use cases (or add these to the backlog if you are using that approach)
  • Allow developers to write code and unit tests simultaneously (not write code based on tests... things change)
  • Implement a monitor to identify code base changes
  • When things change, run the unit tests/macros to confirm things are functioning properly.
